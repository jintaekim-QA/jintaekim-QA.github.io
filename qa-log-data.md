# 📊 QA / Log / Data

라이브 스트리밍 서비스의 **운영형 품질 관리 체계**를 구축하며  
로그 · 데이터 기반 검증을 통해 **사용자 경험·성능·안정성**을 체계적으로 개선한 프로젝트들입니다.

---

## 📘 1. QA 프로젝트

### ✔️ 운영 기반 QA 체계 정립
- 기능/비기능/호환성 테스트 전반 수행  
- 신규 기능 QA 전략 수립 및 테스트 시나리오 표준화  
- 릴리즈 전 회귀 테스트 프로세스 고도화  
- 외주 QA팀 구성 → 테스트 가이드·이슈 관리 프로세스 표준화  
- 실서비스 관점의 품질 기준(UX·성능·안정성) 정립  

### ✔️ 주요 성과
- 릴리즈 잔여 이슈 감소  
- 회귀 테스트 안정화  
- QA 프로세스 리딩 및 협업 구조 강화  

---

## 🔍 2. 로그 기반 품질 인사이트

### ✔️ Firebase / Kibana 기반 사용자 로그 분석
- 실사용자 행동 로그 기반 기능별 진입/이탈 흐름 분석  
- 오류 로그 패턴 추출 → 장애 재현 및 근본 원인 규명  
- 네트워크 상태·단말 성능·UX 흐름에 따른 사용자 불편 지점 탐색  
- CS / VOC와 로그 데이터를 매칭해 재발 방지 로직 도출  

### ✔️ 주요 인사이트 사례
- 특정 기능 뎁스가 깊어 사용률 저하 → 구조 개선 제안  
- 네트워크 약한 환경에서 특정 API 지연 발생 → 로딩 처리 개선  
- 초기 실행 시 CPU burst → Splash 구조·리소스 초기화 개선  

---

## 📊 3. 데이터 기반 품질 검증 체계 구축

### ✔️ 품질 메트릭 자동 수집 시스템 구축
- CPU / RAM / 배터리 / 네트워크 상태 자동 수집  
- 방송 송출/시청 중 성능·안정성 지표 모니터링  
- 로그 기반 지표(Drop Frame, Bitrate, Join Time 등) 정량 검증  
- 모든 데이터를 시각화(Python + Matplotlib) → 리포트 자동 생성

### ✔️ 테스트 효율화
- 병목 파트 자동 알림 → 이슈 조기 발견  
- 운영 단계에서 발생하는 품질 저하 트렌드 자동 감지  
- 네트워크별(Baseline / 5G / LTE / 제한모드) 성능 차이 분석

---

## 🎨 4. UX / 사용성 품질 개선 프로젝트

### ✔️ 로그 기반 UX 흐름 개선
- 사용자 행동 경로 분석 → 불필요한 클릭·뎁스 축소  
- 주요 진입 포인트 집중 분석해 접근성 강화  
- TV/모바일 사용자 시야·조작 패턴 기반 인터랙션 최적화  

### ✔️ UX 개선 후 성과
- 메뉴 뎁스 단축 → 기능 도달 시간 단축  
- 긍정 리뷰 키워드 20% 증가  
- 주요 기능 사용률 상승
