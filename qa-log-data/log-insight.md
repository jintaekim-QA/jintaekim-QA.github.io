# 📊 로그 기반 품질 인사이트(Log Insight)

로그는 기능 정상 동작 여부뿐 아니라 **사용자 행동·품질 이벤트·결함 원인**을 직접적으로 보여주는  
가장 강력한 품질 지표입니다.  
본 문서는 SOOP 프로젝트에서 실제로 운영한 **로그 기반 품질 분석 체계**를 정리합니다.

---

## 🔎 1. 로그 수집 전략

| 항목 | 내용 |
| --- | --- |
| **수집 시스템** | Firebase, Kibana, Internal Log Pipeline |
| **로그 구조화 방식** | 기능 단위 Key-Value 설계 (event_id, page, action, metadata) |
| **주요 수집 포인트** | 클릭/전환/이탈, API Response, 오류/예외, Buffer/Crash 이벤트 |
| **목적** | 사용자 흐름 분석 + 품질 이벤트 조기 탐지 + RCA(근본 원인) 분석 |

> 핵심 목표: **“로그를 품질의 언어로 번역하는 것”**

---

## 📈 2. 사용자 행동 분석(Top Flow & Usage Pattern)

로그를 통해 **사용자 행동 흐름을 정량화**해 테스트 우선순위를 최적화했습니다.

### ✔️ 주요 분석 항목
- 주요 유입 경로(DAU 기준 Top Flow)
- 기능별 사용 빈도 통계
- 이탈·중단 지점 자동 리포트
- 비정상 클릭 루프 탐지

### ✔️ 효과
- 사용 빈도가 높은 기능부터 테스트 집중  
- 특정 OS·네트워크 환경에서 발생하는 사용자 오류 즉시 탐지  
- UX 개선 근거 데이터 확보

---

## 🚨 3. 품질 이슈 자동 탐지 (Buffering / Crash / API Failure)

| 지표 | 활용 방식 | 효과 |
| --- | --- | --- |
| **Buffering Log** | 이벤트 발생 시각·네트워크 상태·재생 구간 매칭 | 스트리밍 지연 지점 파악 → 버퍼링 원인 구간 분석 |
| **Crash Log** | Exception, Stack, Build Version 자동 수집 | 크래시 재현 불가 이슈 감소 / 릴리즈 게이트 운영 |
| **API Failure** | status code / latency / payload diff 비교 | API 스키마 변경 및 응답 지연 즉시 탐지 |

> 자동화 스크립트로 API Response diff 체크하여  
> 서버 변경이 클라이언트에 미치는 영향을 **릴리즈 전 자동 검증**.

---

## 🧪 4. 신규 기능 릴리즈 로그 트래킹

### 📌 운영 사이클
1. 신규 기능별 로그 포인트 설계  
2. 릴리즈 빌드에 삽입 → QA 중 실시간 수집  
3. Event 기반 전환율·유입률·오류율 자동 리포트  
4. UX 문제·버그 패턴 발견 시 즉시 이슈화

### 📌 효과
- 신규 기능 안정성 검증 속도 향상  
- UX 개선 포인트 즉시 발견  
- “사용자가 실제로 어떻게 쓰는가?”를 데이터로 파악

---

## 📉 5. 로그 기반 RCA (Root Cause Analysis)

로그 + API + Network + Device 정보를 매칭해  
재현이 어려운 문제도 근본 원인을 빠르게 확인했습니다.

### 활용 예시
- 특정 단말군에서만 발생하는 Crash 원인 식별  
- API timeout 발생 시 OS·네트워크·Payload 비교  
- 이탈 사용자 다수 발생 기능의 UX 개선 근거 확보

---

## 📊 6. 시각화 & 리포팅 자동화

| 도구 | 목적 |
| --- | --- |
| **Kibana Dashboard** | 실시간 품질 지표 확인(Crash, Buffer, API Error) |
| **Plotly 기반 리포트 자동화** | 매일/매주 품질 변화 그래프 자동 생성 |
| **Slack·Telegram Bot** | 임계치 초과 시 알림 |

### 성과
- QA·Dev 간 커뮤니케이션 속도 증가  
- 품질 상태를 “데이터로 말하는” 체계 구축  
- QA 대응 시간 약 **40% 단축**

---

## 🏁 요약

> 로그는 단순 부가정보가 아닌  
> **서비스 품질을 설계·관리하는 핵심 자산**입니다.  
> 로그 기반 QA를 통해 기능 안정성 뿐 아니라  
> 사용자 경험 품질까지 실시간으로 관리할 수 있는 체계를 구축했습니다.
