# 🧪 QA 프로젝트
데이터 기반 품질 설계와 운영형 QA 체계를 구축하며  
**안정성 · 성능 · 자동화 · 로그 분석을 통합한 품질 전략**을 운영한 사례입니다.

---

## 📌 1) 프로젝트 개요
| 항목 | 내용 |
|-----|------|
| **프로젝트명** | SOOP 라이브 스트리밍 플랫폼 |
| **기간** | 2023.06 ~ 현재 |
| **핵심 목표** | 기능/성능 안정성 확보, 자동화 고도화, 로그 기반 품질 분석 |
| **역할** | QA 리딩(기획·검증·데이터 분석) |

---

## 📌 2) 품질 전략 체계
SOOP 프로젝트에서는 QA 활동을 아래 3단계로 설계했습니다:

1️⃣ **기능 안정성 확보**  
2️⃣ **자동화 기반 효율화**  
3️⃣ **데이터 기반 품질 인사이트 확보**

→ 단순 테스트가 아닌 “운영형 품질 체계”로 전환하여 서비스 신뢰도를 높였습니다.

---

## 📊 3) 품질 지표 기반 모니터링 (Crash & ANR)

### **비정상 종료 / ANR 추이**
![crash_anr](../assets/images/crash-anr-trend.png)

| 지표 | 결과 | 의미 |
|------|-------|--------|
| **비정상 종료 미발생률** | 99.74% | 안정성 확보 |
| Android 평균 | 99.69% | 다양한 단말에서도 안정적 |
| iOS 평균 | 99.68% | 플랫폼 일관성 |
| TV 평균 | 98.68% | 멀티 플랫폼 품질 균형 |
| **ANR 지표** | 누적 16만 / 일 1,500 | 실시간 추적 & 조기 대응 |

> 지표는 QA 주간 리포트 및 CI 품질 게이트로 자동 연동되어  
> 빌드 승인 여부를 **데이터 기반**으로 결정하도록 운영하였습니다.

## 🧩 4) 주요 활동

### ✔ 기능·UI/UX 검증
- 사용자 중심 시나리오 기반 테스트 설계
- UX 개선 인사이트 도출 → 긍정 리뷰 20% 증가

### ✔ 자동화 구축
- Appium 기반 프레임워크 설계
- CI/CD 연동 회귀 자동화 → 커버리지 30% 확대

### ✔ API 검증
- Fiddler 기반 캡처 → API 구조 자동 비교 스크립트 운영

### ✔ 성능 분석
- CPU·RAM·배터리·네트워크 실시간 수집/리포트 자동화

### ✔ 로그 기반 사용자 분석
- Kibana/Firebase로 Top Flow 분석 & 품질 리스크 탐지

## 🎯 5) 성과
- 글로벌 앱 안정성 확보 → **크래시/ANR 지표 정상화**
- 자동화 커버리지 30% → 배포 리스크 대폭 감소
- 로그 기반 사용자 행동 분석 → UX 개선 근거 확보
