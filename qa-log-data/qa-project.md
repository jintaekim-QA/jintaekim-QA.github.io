# 📌 QA 프로젝트 개요  
**모바일 라이브 스트리밍 플랫폼 SOOP – 품질 운영 체계 구축**

이 문서는 SOOP 모바일 앱(Android / iOS / TV)의 품질 운영 과정에서 수행한  
**프로젝트 수준의 QA 활동, 자동화, 로그 분석, 성능 검증, UX 개선 활동**을 정리한 포트폴리오 문서입니다.

---

# 🎯 1. 프로젝트 목적

| 항목 | 내용 |
| --- | --- |
| **서비스명** | SOOP 라이브 스트리밍 플랫폼 |
| **기간** | 2023.06 ~ 현재 |
| **QA 목표** | 기능 안정성 확보 / 자동화 고도화 / 성능 지표 수집 / 로그 기반 품질 인사이트 확보 |
| **담당 역할** | 모바일 QA 총괄, 자동화 구축, 글로벌 런칭 리딩, 품질 정책·프로세스 설계 |

> “단순 테스트가 아닌, **품질 데이터를 기반으로 서비스 신뢰를 설계하는 QA**”를 목표로 품질 운영 체계를 구축했습니다.

---

# 🔍 2. 주요 수행 업무

## ✔️ (1) 기능 안정성 검증  
- 핵심 사용자 플로우(UI·Login·Live 방송·VOD 등)에 대한 시나리오 기반 검증  
- Android/iOS/TV 멀티 디바이스·OS 조합 테스트  
- 신규 기능별 테스트 계획 / 정의 / 리스크 분석  

**성과:** 기능 정상 동작률 상승, 장애 재발률 감소

---

## ✔️ (2) 자동화 구축 & 회귀 테스트 운영  
- Python + Appium 기반 모듈형 자동화 프레임워크 설계  
- View/Test 분리 구조, 공통 action 집약(`QaClass`)  
- GitLab CI/CD와 연동해 **무인 회귀 테스트 파이프라인** 운영  
- APK 자동 설치 → 자동 실행 → 결과 리포트 생성  

**성과:**  
- 자동화 커버리지 **30% 증가**  
- 릴리즈 직전 잔존 결함 사전 발견률 증가  
- 야간/주말 **24h 무인 테스트** 운영 가능  

---

## ✔️ (3) 성능 지표 자동 수집  
- ADB 기반 CPU / RAM / Battery / Network 수집 스크립트 개발  
- 방송 중 실시간 모니터링 + Plotly 그래프 자동 생성  
- 임계치 초과 시 Slack / 메신저로 알림  

**성과:** 성능 이슈 사전 탐지 → 사용자 불만 감소, 최적화 근거 확보

---

## ✔️ (4) API & 로그 기반 분석  
- Fiddler 기반 API 요청·응답 캡처 자동 분석  
- Firebase/Kibana 로그 기반 사용자 행동 분석  
- 오류 로그 기반 Crash/ANR/버퍼링 지표 모니터링  
- Top Flow 시각화로 품질 리스크 우선순위 재설정  

**성과:**  
- 재현 불가 이슈의 근본 원인 파악 속도 증가  
- UX 개선 근거 확보  
- API 파라미터 불일치 및 숨겨진 오류 다수 검출

---

## ✔️ (5) 글로벌 런칭 QA 리딩 (2024)
- 글로벌 SOOP 버전(Android/iOS) 런칭 QA 총괄  
- 베트남 협력 QA팀 10명 리딩  
- 국가·OS·단말·네트워크 환경별 테스트 플랜 수립  
- 런칭 후 안정화 리포트 운영 / 자동화 도입  

**성과:**  
- **일정 내 글로벌 런칭 성공**  
- 협력 QA 프로세스 정착  
- 성능 안정화 및 자동화 기반 유지보수 체계 확립

---

# 📊 3. 안정성 지표 (Crash & ANR)

**(예시 그래프 이미지 첨부 가능 → /assets/… 에 업로드하면 자동 표시 가능)**  

- 전체 비정상 종료 미발생률: **99.74%**  
- Android 평균: **99.69%**  
- iOS 평균: **99.68%**  
- Android TV: **98.68%**  
- ANR: 일평균 1,500건 → 지표 기반 감지·대응 체계 구축  

> 안정성 지표는 QA 리포트와 CI 파이프라인 게이트로 연동되어 빌드 승인 판단 근거로 사용.

# 📈 4. 품질 인사이트 사례

### ✔️ 사용자 행동 로그 기반 기능 리스크 분석  
- 클릭 이벤트 기반 사용자 주요 흐름 분석  
- 유입률/전환률 데이터 기반 UI 개선  
- 버퍼링 및 오류 발생 지점 시각화  

### ✔️ 성능 이슈 원인 분석  
- 특정 단말군에서 CPU spike 문제 발견 → 개발팀과 협력하여 개선  
- 네트워크 환경별 방송 지연 문제 해결  

### ✔️ 장애 재발 방지 프로세스 구축  
- 이벤트 로그 추적 → API 매칭 → RCA 도출  
- Jira 기반 장애 재발 방지 Task 관리  

---

# 🧩 5. 종합 인사이트

> QA는 단순한 “버그 찾기”가 아니라  
> **서비스 신뢰도를 데이터 기반으로 설계하는 엔지니어링 활동**임을 강조하며  
> 자동화·로그·성능·UX·API 검증이 유기적으로 연결된 **종합 QA 운영 체계**를 구축했습니다.
