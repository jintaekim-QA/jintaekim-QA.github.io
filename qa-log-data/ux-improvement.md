# 🎨 UX / 사용성 품질 개선 프로젝트
서비스 전체 사용 흐름을 데이터 기반으로 분석하고,  
사용자 경험을 저해하는 요소를 제거해 **전환율·기능 접근성·사용자 만족도**를 향상시킨 프로젝트입니다.

---

## 📌 1. 프로젝트 개요

| 항목 | 내용 |
| --- | --- |
| **프로젝트명** | SOOP 모바일 앱 UX 개선 |
| **기간** | 2023–2024 |
| **목표** | 사용자 흐름 단순화, 전환율 개선, 기능 접근성 증대 |
| **사용 데이터** | Firebase Analytics, Kibana 로그, 사용자 행동 데이터 |
| **테스트 환경** | Android/iOS 실디바이스, 사용자 리서치, A/B 테스트 |

> **로그 기반 행동 분석 + UX 구조 개선**을 결합하여  
> 실제 사용자 행동에 최적화된 경험을 설계했습니다.

---

## 🎯 2. 문제 정의 (Pain Point 분석)

로그/사용자 행동 분석을 통해 식별한 주요 문제:

- **화면 뎁스가 깊어 기능 접근이 느림**
- 핵심 기능에 도달하기 위한 **탭 이동 수가 많음**
- 사용자 전환 로그에서 **이탈 비율 높은 구간** 존재
- 일부 기능의 위치가 **사용자 기대 mental model과 불일치**
- TV/모바일 간 UX 패턴 차이로 **일관성 부족**

---

## 🔍 3. 분석 방식

### ① Top Flow 분석 (주요 사용 경로)
- 클릭/전환/이탈 로그 기반으로 실제 사용 흐름 시각화
- 사용률 높은 기능/낮은 기능 자동 통계화
- 경로별 평균 체류 시간 분석

### ② UX Metric 기반 문제 추출
- 기능 진입 성공률
- 클릭 대비 전환율(CTR → CVR)
- 버튼 가시성 분석
- 화면 체류 시간
- 스크롤 깊이/버튼 위치 Heatmap

### ③ 사용자 VOC + 이슈 로그 매칭
- VOC에서 불편 지점 수집
- Firebase/Kibana 이슈 로그와 매칭하여 **객관적 근거 확보**

---

## 🛠️ 4. 개선 작업

### 🔹 메뉴 구조 단순화
- 불필요한 메뉴 제거
- 자주 사용하는 기능을 상단 배치
- 핵심 기능 진입 뎁스 **2단계 이하로 축소**

### 🔹 기능 접근 단계 단축
- 기존 4~5단계 접근 → **2단계 진입 가능하도록 구조 재배치**
- UI 요소의 중요도 기반 재배열

### 🔹 버튼 가시성 강화
- 클릭률 낮던 버튼의 위치·색상·텍스트 개선
- “숨겨진 기능”을 사용자 눈높이에 맞춰 재배치

### 🔹 TV/모바일 UX 일관성 확보
- TV UI 특성(거리 기반 UI, 포커스 이동)을 고려한 맞춤 구조
- 모바일-웹-TV 간 공통 UX 원칙 재정립

---

## 🚀 5. 개선 결과

| 지표 | 개선 전 | 개선 후 | 개선 효과 |
| --- | --- | --- | --- |
| **기능 접근 단계** | 평균 4.2 단계 | **2.1 단계** | 접근성 대폭 향상 |
| **주요 기능 사용률** | 기존 대비 낮음 | **+15% 증가** | 행동 패턴 개선 |
| **긍정 리뷰 비율** | 기준치 | **+20% 증가** | UX 만족도 향상 |
| **이탈 구간 감소율** | - | **약 12% 감소** | 전환 흐름 안정화 |

> UX 개선 이후 실제 사용자 리뷰에서  
> “이용하기 쉬워졌다”, “길이 짧아졌다”, “찾기 쉬워졌다”와 같은 긍정 키워드 증가.

## 📝 7. 기여 포인트 정리

- 로그 데이터 기반 **정량적 UX 분석 구조** 직접 설계
- 기능/사용성 개선을 **사용자 행동 데이터 기반**으로 추진
- 개발·기획과 협업하여 **UX 구조 전면 재정비**
- 실제 KPI로 **사용률·접근성·리뷰 지표 모두 상승**

---

## 🧩 8. 마무리

UX 개선은 단순 디자인 작업이 아니라  
**사용자의 행동을 이해하고 데이터로 문제를 정의하는 과정**입니다.

이 경험을 통해  
‘예쁘게 만드는 UX’가 아니라  
**서비스 성공을 위한 전략적 UX 개선**을 수행할 수 있는 역량을 증명했습니다.
